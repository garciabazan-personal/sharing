Concept Glossary #
Seven terms, seven plain-language definitions — with analogies to make them stick and technical notes for the curious.
Foundation

LLM — Large Language Model

An AI system trained on massive amounts of text to understand, generate, and reason about language. It is the core "reasoning engine" behind virtually every modern AI tool — the model you are actually talking to when you use Claude, ChatGPT, or Gemini.

Analogy: Think of an LLM as a consultant who has read virtually everything ever written. They can answer questions, draft documents, explain concepts, and reason through problems — but they need you to bring context and ask good questions.

Examples: Claude (Anthropic), GPT-4o (OpenAI), Gemini (Google). An LLM alone is stateless — no memory between conversations, no ability to take actions unless you layer tools on top.

Interface

CLI — Command Line Interface

A text-based tool you run in a terminal to interact with software. In AI contexts, a CLI like Claude Code lets you have a conversation with an AI model while it can also read your local files, run commands, and make changes to your system — far more powerful than a browser-based chat window.

Analogy: If the LLM is the consultant, the CLI is the private office where you meet. No middleman, no GUI overhead — just you, the model, and full access to your machine and connected tools.

Claude Code is Anthropic's CLI. It gives Claude direct access to your file system, shell, and any connected MCP servers — enabling it to act as a true coding and analysis partner, not just a chat assistant.

Protocol / Integration

MCP — Model Context Protocol

An open standard that lets AI models connect to external tools, data sources, and services in a secure, standardized way. Each MCP server exposes a set of capabilities — like querying BigQuery, reading Jira tickets, or searching internal docs — that the AI can call during a conversation.

Analogy: MCPs are like universal power adapters. The AI is your device; MCP servers are adapters that let it plug into anything — BigQuery, Jira, GitHub, Slack, dbt Cloud — without custom wiring for each connection.

Active MCP servers: DataConcierge (data discovery), BigQuery (querying), Glean (internal search), Atlassian (Jira + Confluence). The AI can call these tools mid-conversation to retrieve live data.

Reusable Capability

Skill

A predefined, reusable instruction set — or prompt template — that encodes how to do a specific task well. Skills are invoked like slash commands (e.g., /generate-dbt-model or /summarize-pr) and give the AI expert-level context for that task without the user having to explain it every time.

Analogy: Skills are like Standard Operating Procedures. Instead of explaining from scratch how to conduct a code review or write a PRD every time, you encode the best approach once — and anyone can invoke it consistently.

In Claude Code, skills are defined in CLAUDE.md or as custom slash commands. They can encode team conventions, reference specific tools, and set output format expectations — making expert behavior repeatable.

Autonomous Actor

Agent

An AI system that goes beyond answering questions — it can take actions, use tools, make decisions, and work toward a goal autonomously over multiple steps. An agent combines an LLM with tools (via MCP), memory, and a feedback loop so it can plan, execute, self-correct, and iterate.

Analogy: If an LLM is a consultant you ask questions, an agent is a consultant you hand a project to. They scope the work, use their tools, check their own results, and come back when done — without you directing every single step.

Key properties: autonomy, tool use, goal-orientation, and error recovery. Claude Code in agentic mode is an agent — it can read, write, run tests, search the web, query databases, and iterate on your behalf.

Specialized Actor

Sub-agent

An agent that operates within a larger agentic system, handling a specific, scoped piece of a bigger task. An orchestrator agent breaks work into pieces and delegates each piece to a sub-agent specialized for that domain. Sub-agents run independently and report results back.

Analogy: Like a project team. The orchestrator is the project manager who breaks down work. Sub-agents are the specialists: one pulls data, one runs analysis, one writes the narrative. Each is capable on their own, but they operate under a common plan.

In Claude Code, the Task tool launches sub-agents: "Explore" for codebase search, "Bash" for terminal commands, "Plan" for architecture decisions. Each runs in an isolated context with its own tool access.

Orchestrated System

Agentic Workflow

A structured, multi-step process where one or more agents autonomously plan, execute, and adapt to complete a complex end-to-end task. Unlike a single agent interaction, a workflow involves sequencing, parallel execution, conditional branching, and human-in-the-loop checkpoints at critical decision points.

Analogy: An agentic workflow is like a fully automated assembly line for knowledge work. Each station (sub-agent or step) does its part and passes results forward. A human approves key checkpoints before the system takes consequential actions.

Typical components: triggering event → orchestrator → parallel sub-agents → tool calls via MCP → state management → human review gate → final output. The key differentiator from a script is the AI's ability to adapt when things go wrong.

Sub-agent
In a strategy workflow, one sub-agent handles competitive research while another synthesizes internal customer feedback — the orchestrator combines both into a final strategy brief.
Sub-agent A pulls Jira themes; Sub-agent B pulls Glean docs; orchestrator merges into roadmap.
In a reporting pipeline, a data retrieval sub-agent runs in parallel with a visualization sub-agent — each focused on its specialty, with the orchestrator assembling the final report.
Sub-agent A queries BigQuery; Sub-agent B writes chart YAML; orchestrator builds the deck.
In a data quality workflow, a schema validation sub-agent runs independently of a freshness-check sub-agent — the orchestrator aggregates findings and decides whether to alert or auto-remediate.
Sub-agent A checks schema drift; Sub-agent B checks SLA violations; orchestrator acts.
In an AutoML pipeline, a feature engineering sub-agent runs in parallel with a hyperparameter search sub-agent — the orchestrator evaluates both outputs and selects the winning configuration.
Sub-agent A generates feature combinations; Sub-agent B runs Optuna; orchestrator picks winner.
